# 历史地图应用 - JSON文件生成规则

## 1. JSON文件格式

历史地图应用使用JSON格式存储历史事件标记数据。以下是标准的JSON文件结构：

```json
{
  "markers": [
    {
      "title": "事件标题",
      "description": "事件描述（可选）",
      "time": 年份（公元前使用负数）,
      "location": "地点（可选）",
      "latitude": 纬度,
      "longitude": 经度
    }
  ]
}
```

## 2. 字段说明

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| title | 字符串 | 是 | 事件标题，将显示在标记点和时间线上 |
| description | 字符串 | 否 | 事件详细描述，将显示在弹窗中 |
| time | 数字 | 是 | 事件发生年份，公元前使用负数，如公元前200年表示为-200 |
| location | 字符串 | 否 | 事件发生地点，将显示在时间线和弹窗中 |
| latitude | 数字 | 是 | 事件发生地点的纬度 |
| longitude | 数字 | 是 | 事件发生地点的经度 |

## 4. 数据准备技巧

1. **经纬度获取**：可以使用Google Maps等地图服务获取地点的经纬度
2. **时间格式**：确保时间格式正确，公元前使用负数
3. **事件排序**：应用会自动按时间顺序排序事件，无需手动排序
4. **重复坐标**：如果多个事件发生在同一地点，应用会自动为标记点添加偏移，避免重叠
5. **数据量**：建议单个JSON文件不要包含过多标记点，以保持良好的性能

## 5. 使用方法

1. 创建符合上述格式的JSON文件
2. 在应用中点击"导入JSON数据"按钮
3. 选择创建的JSON文件
4. 应用会自动加载并显示所有标记点

## 6. 注意事项

- 确保JSON格式正确，否则会导入失败
- 经纬度值应使用正确的数值格式（如：35.6892，而不是"35.6892"）
- 时间值应使用数字格式（如：1979，而不是"1979"）
- 对于大量标记点，建议分批导入，以提高性能

## 7. 常见问题

### Q: 导入后标记点不显示？
A: 检查JSON格式是否正确，特别是经纬度和时间字段是否为数字格式

### Q: 标记点重叠在一起？
A: 应用会自动处理重复坐标，为标记点添加偏移

### Q: 时间线显示顺序不正确？
A: 应用会自动按时间顺序排序，确保time字段格式正确

### Q: 地点信息不显示？
A: 检查location字段是否正确设置

## 8. 推荐工具

- **JSON验证器**：使用在线JSON验证工具确保文件格式正确
- **经纬度查询**：使用Google Maps或其他地图服务获取准确的经纬度
- **历史资料**：参考可靠的历史资料确保事件时间和地点的准确性
